# CGSS Qwen2.5 Prompt Matrix Results (2026-05-08)

This directory adds the CGSS experiments for the final project. Existing files under `results/` were not modified or replaced.

## Main Results

Lower KL and WD are better. Delta is `fine_tuned - baseline`, so negative values mean the fine-tuned model is closer to the CGSS target distribution.

## Prompt Types

- QA: the subgroup condition is represented as a prior survey question-answer pair before the target question.
- BIO: the subgroup condition is written as a first-person biography/self-description before the target question.
- PORTRAY: the model is directly asked to assume a persona matching the subgroup condition.

## Directory Layout

- `summary.csv` and `summary.json`: aggregated KL/WD table for the six model-prompt settings.
- `csv/baseline/`: zero-shot or base-model evaluation outputs.
- `csv/fine_tuned/`: fine-tuned model evaluation outputs.
- `csv/paired_compare/`: paired baseline-vs-fine-tuned comparison CSVs.
- `figures/`: comparison plots generated by `compare_eval_csv.py`.
- `metrics/`: training metric JSON files from the fine-tuning output folders.
- `logs/`: run logs for the prompt-matrix experiments.
- `code/`: key scripts used to run fine-tuning, evaluation, and comparison.

## Run Notes

- Models: `Qwen2.5-0.5B` and `Qwen2.5-7B`.
- Dataset: CGSS converted into the project `opnqa_cgss_steering_dataset` format.
- Fine-tuning: QLoRA 4-bit, LoRA `r=16`, `alpha=64`, `dropout=0.05`, context length 1024, CE loss, 3 epochs.
- Evaluation: CGSS `opnqa_{QA,BIO,PORTRAY}_test.csv`; metrics are average KL divergence and Wasserstein distance.
- Important hygiene fix: `outputs/peft_checkpointing` was moved aside before each prompt run, because `run_finetune.py` can otherwise resume an adapter from a previous prompt and contaminate the comparison.
- The valid rerun for `Qwen2.5-7B + BIO` is `outputs/colab_qlora4bit_qwen2.5-7b_cgss_BIO_ce_20260508_204109`.
